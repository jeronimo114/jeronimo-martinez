APIs Gratuitas para tu Bot

Conecta tu agente al mundo real

Que es una API?

Tu bot envia una peticion → recibe datos → los usa en su respuesta

GET Request

La mayoria de estas APIs usan peticiones GET: simplemente visitas una URL y te devuelve datos en formato JSON. Es como abrir una pagina web, pero en vez de HTML recibes datos estructurados.

Como usar en n8n

Usamos el nodo HTTP Request Tool (no el HTTP Request normal)

HTTP Request Tool

Importante

El nodo HTTP Request Tool permite que el agente DECIDA cuando llamar a la API. El agente lee la pregunta del usuario, decide si necesita datos externos, y hace la llamada automaticamente.

API 1

icanhazdadjoke.com

Chistes en ingles (dad jokes)

https://icanhazdadjoke.com/
Header → Accept: application/json
Response JSON
{ "id": "R7UfaahV...", "joke": "What do you call a...", "status": 200 }

Header requerido

Esta API requiere el header Accept: application/json para devolver JSON. Sin el, devuelve texto plano.

Ejercicio 1

Ejercicio: Chistes

Pasos

  1. Agrega un nodo HTTP Request Tool al canvas
  2. Conectalo al puerto Tool del AI Agent
  3. URL: https://icanhazdadjoke.com/
  4. Method: GET
  5. Agrega Header: Accept = application/json
  6. Prueba: "Cuentame un chiste"

Verificacion

El agente deberia responder con un chiste diferente cada vez. Revisa los logs para confirmar que se hizo la llamada HTTP.

API 2

api.agify.io

Adivina la edad basandose en el nombre

https://api.agify.io?name=Juan
Response JSON
{ "count": 1234, "name": "Juan", "age": 45 }

Parametro dinamico

Puedes cambiar "Juan" por cualquier nombre. El agente puede construir la URL con el nombre que el usuario mencione.

Ejercicio 2

Ejercicio: Edad

Pasos

  1. Agrega un nuevo HTTP Request Tool
  2. Conectalo al puerto Tool del AI Agent
  3. URL: https://api.agify.io?name=Juan
  4. El agente debe extraer el nombre del usuario y pasarlo como parametro
  5. Prueba: "Que edad crees que tengo? Me llamo Carlos"
  6. Verifica que responda con una edad estimada

URL dinamica

El agente puede construir la URL dinamicamente reemplazando el nombre en el parametro ?name= segun lo que diga el usuario.

API 3

dog.ceo/api

Fotos aleatorias de perros

https://dog.ceo/api/breeds/image/random
Response JSON
{ "message": "https://images.dog.ceo/breeds/...", "status": "success" }

Imagen aleatoria

Devuelve una URL de imagen aleatoria de un perro. Cada llamada genera una imagen diferente.

  4. Method: GET
  5. Prueba: "Mandame una foto de un perro"
  6. El bot deberia devolver el link de la imagen

Telegram

Si tu bot esta conectado a Telegram, puede mostrar imagenes directamente si envias la URL. El usuario vera la foto del perro en el chat.

API 4

Open-Meteo

Clima en tiempo real (sin API key)

https://api.open-meteo.com/v1/forecast?latitude=6.25&longitude=-75.56¤t_weather=true
Response JSON (parcial)
{ "current_weather": { "temperature": 22.5, "windspeed": 8.3, "weathercode": 1 } }

Coordenadas

Medellin: lat=6.25, lon=-75.56 | Bogota: lat=4.71, lon=-74.07 | CDMX: lat=19.43, lon=-99.13. Cambia latitude y longitude para otras ciudades.

API 5

zenquotes.io

Frases celebres e inspiradoras

https://zenquotes.io/api/random
Response JSON
[{ "q": "The only way to do great work...", "a": "Steve Jobs", "h": "..." }]

Bot motivacional

Perfecta para un bot motivacional o coach. El campo "q" es la frase y "a" es el autor.

Formato ideal

Pide al agente en el System Prompt que formatee como: "La frase..." - Autor

API 6

restcountries.com

Datos completos de cualquier pais

https://restcountries.com/v3.1/name/colombia
Response JSON (parcial)
{ "capital": ["Bogota"], "population": 50882891, "languages": { "spa": "Spanish" }, "currencies": { "COP": { "name": "Colombian peso" } } }

Uso ideal

Ideal para un asistente de viajes o cultura general. Devuelve capital, poblacion, idiomas, moneda, bandera y mucho mas.

Multiples APIs

Puedes conectar VARIAS APIs al mismo agente

Como funciona

Cada HTTP Request Tool es independiente. El agente decide cual usar segun la pregunta del usuario. Puedes tener 2, 3 o mas APIs conectadas al mismo tiempo.

Reto

  1. Conecta al menos 2 APIs diferentes al mismo agente
  2. Prueba haciendo preguntas variadas: "Cuentame un chiste y despues dime el clima"
  3. Verifica que el agente use la API correcta para cada pregunta
