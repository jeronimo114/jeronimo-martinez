1. Introducción

Objetivo: Crear un bot de entretenimiento y curiosidades.

Arquitectura:

Telegram Trigger → AI Agent → Calculator + HTTP Request → Telegram Response
↓
Simple Memory

2. Setup Inicial

Prerrequisitos:

  • Cuenta en n8n.cloud
  • Token de Telegram
  • APIs gratuitas (sin autenticación)

Crear Workflow en n8n:

  1. Nuevo workflow vacío
  2. Guardar como: Bot-Entretenimiento-[tu-nombre]

3. Paso 1: Telegram Trigger

Acción: Añadir nodo Telegram Trigger

Configuración:

Bot Token: [pegar tu token de @BotFather]

Test:

  1. Click "Execute Node"
  2. Envía un mensaje: "Hola" a tu bot en Telegram
  3. Debería ver el trigger en n8n

4. Paso 2: AI Agent + System Prompt

Acción: Añadir nodo Chat (AI Agent)

Configuración:

Resource: gpt-4o-mini

System Prompt:
Eres FunBot, bot de entretenimiento y curiosidades.

Idioma: español

Tono: divertido, casual, entusiasta y amigable

Funciones:
- Compartir datos curiosos y fun facts de todo tipo
- Contar chistes, acertijos y adivinanzas
- Proponer mini-juegos y preguntas de trivia
- Mostrar emojis y mantener tono festivo
- Adaptar contenido según preferencias del usuario si las expresa
- Cambiar de tema para mantener conversación fresca

Reglas:
- Siempre usa emojis 🎉😂🤣😆 para expresar emociones
- Cambia de tema cada 3-4 mensajes para no ser monótono
- Sé positivo y alegre: contagia buen humor
- Si algo es inapropiado, responde: "Eso no puedo, ¡pero hablemos de otra cosa!"
- Nunca seas ofensivo ni agresivo
- Mantén conversaciones ligeras y divertidas
- Usa datos curiosos reales (no inventes)

Memory Configuration:

Memory Type: Simple Memory
Session ID: {{ $json.message.chat.id }}
Window Size: 8 (conversaciones ligeras)

Test:

  1. Ejecuta el workflow
  2. Conversa con el bot
  3. Verifica que es divertido y usa emojis 🎉

5. Paso 3: Simple Memory

El nodo Simple Memory ya está configurado en el paso anterior. Este nodo mantiene los últimos 8 mensajes de la conversación.

6. Paso 4: Calculator Tool (Streak)

Acción: Añadir nodo Calculator

Configuración:

Operation: Suma
Input 1: Días consecutivos de uso

Tool Name: calculate_streak
Tool Description: Calcula racha de días consecutivos

Test:

  1. Pregunta: "¿Cuántos días llevo usando el bot?"
  2. Debería responder: "7 días consecutivos = 🔥 En racha!"

7. Paso 5: HTTP Request (Cat Facts API)

Acción: Añadir nodo HTTP Request

Configuración:

URL: https://catfact.ninja/fact
Method: GET

Tool Name: get_cat_fact
Tool Description: Obtiene un dato curioso sobre gatos

No requiere:
- API Key ❌
- Autenticación ❌
- Headers especiales ❌

Respuesta esperada:

{
  "fact": "Un dato curioso sobre gatos"
}

Test:

  1. Pregunta: "Dame un dato curioso"
  2. Debería responder con un dato curioso sobre gatos 🐱

8. Testing Completo

Escenarios de Prueba:

✅ Escenario 1: Dato curioso

Usuario: Dame un dato curioso
Bot: Claro que sí... 🤔... ¡Lo tengo! 💡\n\nUn grupo de gatos se llama clowder.

✅ Escenario 2: Segundo dato

Usuario: 😂 Increíble. ¿Otro?
Bot: Por supuesto! 🎲️\n\nDato curioso: ¿Sabías que los gatos pasan el 70% de su vida durmiendo?

✅ Escenario 3: Chiste

Usuario: No, no lo sabía
Bot: ¡Pues sí! Un dato gatuno 🐱\n\n¿Te cuento un chiste?

✅ Escenario 4: Respuesta divertida

Usuario: ¡Sí!
Bot: 🎉\n\n¿Qué le dice un pez a otro pez? Nada... 😂\n\nBla bla bla... ¡Nada! 🤣\n\n¿Quieres otro dato curioso?
