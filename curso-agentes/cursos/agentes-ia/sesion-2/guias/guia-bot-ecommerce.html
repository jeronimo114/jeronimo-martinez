1. Introducción

Objetivo: Crear un bot de soporte al cliente para e-commerce.

Arquitectura:

Telegram Trigger → AI Agent → Calculator + HTTP Request → Telegram Response
↓
Simple Memory

2. Setup Inicial

Prerrequisitos:

  • Cuenta en n8n.cloud
  • Token de Telegram
  • Datos de productos (simulados en JSON)

Crear Workflow en n8n:

  1. Nuevo workflow vacío
  2. Guardar como: Bot-Ecommerce-[tu-nombre]

3. Paso 1: Telegram Trigger

Acción: Añadir nodo Telegram Trigger

Configuración:

Bot Token: [pegar tu token de @BotFather]

Test:

  1. Click "Execute Node"
  2. Envía un mensaje: "Hola" a tu bot en Telegram
  3. Debería ver el trigger en n8n

4. Paso 2: AI Agent + System Prompt

Acción: Añadir nodo Chat (AI Agent)

Configuración:

Resource: gpt-4o-mini

System Prompt:
Eres ECommerceBot, especialista en productos de nuestra tienda online.

Idioma: español

Tono: empático, paciente y resolutivo

Funciones:
- Consultar estado de pedido
- Informar sobre productos y características
- Procesar solicitudes de devolución
- Recomendar productos basados en necesidades
- Calcular precios con descuentos

Reglas:
- Siempre pide número de pedido antes de dar información de estado
- Para devoluciones, primero verifica política de devolución (30 días)
- Aplica descuento del 15% para nuevos clientes
- Si el usuario pregunta por un producto que no existe, sugiere alternativas
- Nunca prometas fechas de entrega, solo "2-3 días hábiles"
- Si no tienes información, sé honesto: "Lo siento, no tengo esa información"

Memory Configuration:

Memory Type: Simple Memory
Session ID: {{ $json.message.chat.id }}
Window Size: 10

Test:

  1. Ejecuta el workflow
  2. Conversa con el bot
  3. Verifica que recuerda información de mensajes anteriores

5. Paso 3: Simple Memory

El nodo Simple Memory ya está configurado en el paso anterior. Este nodo mantiene los últimos 10 mensajes de la conversación.

6. Paso 4: Calculator Tool (Descuentos)

Acción: Añadir nodo Calculator

Configuración:

Operation: Calcular descuento
Formula: Precio × (1 - PorcentajeDescuento / 100)
Input 1: Precio del producto
Input 2: Porcentaje de descuento (ej: 15)

Tool Name: calculate_discount
Tool Description: Calcula precio con descuento aplicado

Test:

  1. Pregunta: "¿Cuánto cuesta el iPhone 15 con 15% de descuento?"
  2. Debería responder: "$849.15" (999 - 15% = 849.15)

7. Paso 5: HTTP Request (Stock)

Acción: Añadir nodo HTTP Request

Configuración:

Method: GET
URL: /api/products/iphone15

Tool Name: check_stock
Tool Description: Consulta stock de producto

Opción: JSON Simulado (más fácil para demo)

Crear nodo "Set" con este JSON como valor inicial:
{
  "products": {
    "iphone15": { "name": "iPhone 15", "stock": 15, "price": 999 },
    "macbook": { "name": "MacBook Pro", "stock": 5, "price": 1999 },
    "airpods": { "name": "AirPods Pro", "stock": 30, "price": 249 }
  }
}

Tool Name: get_product_info
Tool Description: Obtiene información de producto (simulado)

Test:

  1. Pregunta: "¿Tienen el iPhone 15?"
  2. Debería responder: "Sí, 15 unidades en stock, precio: $999"

8. Testing Completo

Escenarios de Prueba:

✅ Escenario 1: Consulta de producto

Usuario: ¿Tienen el iPhone 15?
Bot: Sí, 15 unidades en stock, precio: $999

✅ Escenario 2: Cálculo con descuento

Usuario: ¿Cuánto con el 15% de descuento para nuevos clientes?
Bot: Aplicando descuento... $999 - 15% = $849.15

✅ Escenario 3: Producto sin stock

Usuario: ¿Tienen el MacBook Pro?
Bot: Lo siento, el MacBook Pro está agotado. Solo quedan 5 unidades.

✅ Escenario 4: Segunda pregunta con memoria

Usuario: ¿El iPhone 15 tiene buena batería?
Bot: [Debe recordar que estábamos hablando del iPhone 15]
