1. Introducción

Objetivo: Crear un bot de reservas de restaurantes/eventos.

Arquitectura:

Telegram Trigger → AI Agent → Calculator + HTTP Request → Telegram Response
↓
Simple Memory

2. Setup Inicial

Prerrequisitos:

  • Cuenta en n8n.cloud
  • Token de Telegram (via @BotFather)
  • API Key de OpenAI

Crear Workflow en n8n:

  1. Nuevo workflow vacío
  2. Guardar como: Bot-Reservas-[tu-nombre]

3. Paso 1: Telegram Trigger

Acción: Añadir nodo Telegram Trigger

Configuración:

Bot Token: [pegar tu token de @BotFather]

Test:

  1. Click "Execute Node"
  2. Envía un mensaje: "Hola" a tu bot en Telegram
  3. Debería ver el trigger en n8n

4. Paso 2: AI Agent + System Prompt

Acción: Añadir nodo Chat (AI Agent) después del trigger

Configuración:

Resource: gpt-4o-mini (recomendado - barato y rápido)

System Prompt:
Eres ReservaBot, asistente de reservas de restaurantes y eventos.

Idioma: siempre responde en español.

Tono: amable, profesional y servicial.

Funciones:
- Reservar mesas y citas
- Consultar disponibilidad
- Confirmar reservas con nombre

Reglas:
- NUNCA reserves sin confirmar antes
- Si el usuario pregunta por disponibilidad general, primero pregunta: fecha, hora, número de personas
- Para cancelaciones, pide: número de reserva

Memory Configuration:

Memory Type: Simple Memory
Session ID: {{ $json.message.chat.id }}
Window Size: 10 (últimos 10 mensajes)

Test:

  1. Ejecuta el workflow
  2. Conversa con el bot
  3. Verifica que recuerda información de mensajes anteriores

5. Paso 3: Simple Memory

El nodo Simple Memory ya está configurado en el paso anterior. Este nodo mantiene los últimos 10 mensajes de la conversación, permitiendo al bot recordar contexto de mensajes previos.

6. Paso 4: Calculator Tool

Acción: Añadir nodo Calculator como herramienta del AI Agent

Configuración:

Operation: Multiplicación
Input 1: Número de personas
Input 2: Precio por persona

Tool Name: calculate_total
Tool Description: Calcula el precio total de la reserva

Test:

  1. En el chat, pregunta: "¿Cuánto cuesta para 3 personas a $40 cada una?"
  2. Debería responder: "$120"

7. Paso 5: HTTP Request (Clima)

Acción: Añadir nodo HTTP Request como herramienta del AI Agent

Configuración:

Method: GET
URL: https://api.open-meteo.com/v1/forecast
Query Parameters:
  latitude: 40.4168  (Madrid)
  longitude: -3.7038
  current_weather: true

Tool Name: get_weather
Tool Description: Obtiene el clima actual para una ciudad

Test:

  1. Pregunta: "¿Qué clima hace en Madrid?"
  2. Debería responder con clima actual

8. Testing Completo

Escenarios de Prueba:

✅ Escenario 1: Reserva básica

Usuario: Quiero reservar 3 personas mañana a las 14h
Bot: [Confirmar detalles: fecha, hora, nombre]

✅ Escenario 2: Segunda reserva con memoria

Usuario: Por cierto, cambia a 4 personas
Bot: [Debe recordar conversación anterior + actualizar]

✅ Escenario 3: Consulta de clima antes de reservar

Usuario: ¿Qué clima hace?
Bot: [Consultar API Open-Meteo]

✅ Escenario 4: Cálculo de precio

Usuario: ¿Cuánto cuestan 5 personas a $25?
Bot: [Usar Calculator: 5 × 25 = $125]
